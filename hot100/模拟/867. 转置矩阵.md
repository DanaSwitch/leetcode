# 转置矩阵

## 一. 问答

for _ in range(n) 中的_是什么意思



在 Python 中，`_`（下划线）是一个**合法的变量名**，但在这种上下文里，它有一个约定俗成的特殊含义：**“我不关心这个变量的值”**。

具体到 `for _ in range(n)`，它的意思是：

> **“请循环 n 次，但我根本不需要用到当前的循环索引（比如 0, 1, 2...）。”**

------

### 1. 为什么要用 `_`？

#### 为了代码的可读性

当你看到 `for i in range(n)` 时，你的潜意识会认为：“哦，在这个循环里，变量 `i`（比如第几次循环）后面肯定会被用到。”

但是，在你的代码中：

```python
ans = [[0]*m for _ in range(n)]
```

你的目的是：**“我要生成 n 个空行”**。 你并不关心现在是生成第 0 行还是第 10 行，你只关心动作要重复 `n` 次。

使用 `_` 就像是在告诉读代码的人（以及代码检查工具）：**“忽略这里的索引值，我只是为了凑齐次数。”**

### 2. 对比示例

**场景 A：我们需要用到索引 (使用 `i`)** 比如我们要打印：`第 1 天`，`第 2 天`...

```python
for i in range(5):
    print(f"第 {i+1} 天")  # 这里我们需要用到 i 的值
```

**场景 B：我们只需要重复动作 (使用 `_`)** 比如我们要打印 5 次 `Hello`，第几次并不重要。

```python
for _ in range(5):
    print("Hello")      # 这里完全不需要知道当前是第几次
```

### 3. 补充知识

虽然 `_` 看起来很特殊，但在 Python 解释器眼里，**它就是一个普通的变量**。

如果你非要打印它，也是可以的：

```python
for _ in range(3):
    print(_) 
# 输出:
# 0
# 1
# 2
```

只是大家约定俗成不去读取它。

**其他常见用法：** `_` 也常用于元组解包，用来“丢弃”不需要的返回值。

```python
# 假设函数返回 (姓名, 年龄, 身份证号)，我们只需要姓名
name, _, _ = get_person_info() 
```

------

**总结：** 在你的代码 `[[0]*m for _ in range(n)]` 中，`_` 的作用就是做一个**占位符**，表示单纯地循环 `n` 次来构建矩阵的行。
