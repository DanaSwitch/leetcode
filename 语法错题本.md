### 打印机队列

>n = input().split()
>
>printers = {i: [] for i in range(1, n)}

这个报错的原因非常直接：`n = input().split()` 这一行代码读入后，`n` 变成了一个**列表**（例如 `['7']`），而 `range()` 函数只能接受一个**整数**（如 `7`）。

1. **数据类型错误**：`n = input().split()` 会返回列表。应改为 `n = int(input())`。
2. **`range` 范围错误**：你的打印机编号是 1 到 5，`range(1, n)` 如果 `n` 是 5，只会循环到 4。应改为 `range(1, 6)`。
3. **函数调用混淆**：
   - `printers(p)` 是在尝试“调用”字典，而不是“访问”字典。字典访问应该用方括号：`printers[p]`。
   - `heapq.heappush(printers, ...)`：这里你应该传入的是特定打印机的列表，即 `printers[p]`。
4. **数据类型统一**：输入进来的 `p` 和 `num` 默认是字符串，必须用 `int()` 转换，否则无法进行数值比较或作为字典索引。

### 比赛评分

#### 1. 输入处理错误（最严重）

你使用了 `M, N = input().split(',')`。

- **问题**：`input()` 返回的是一个字符串，`split(',')` 会把它拆分成列表，但此时 `M` 和 `N` **仍然是字符串**。

- **后果**：你在 `if` 条件里写了 `len(M)`，这统计的是字符串“3”（假设输入为 3）的长度，结果是 1，这完全不是你想要的数值大小比较。

- **修正**：必须先转为 `int`。

  ```
  first = input().split(',')
  M, N = int(first[0]), int(first[1])
  ```

#### 2. 循环逻辑错误

你写了 `for _ in range(M):` 来读取评委打分，但是你之前 `M` 是字符串，在 `range()` 里传入字符串会直接报错 `TypeError`。

#### 3. `sort_key` 函数中的拼写与类型错误

```
def sort_key(i):
    key = total[i]  # 错误：total[i] 是一个整数
    for u in range(10, 0, -1):
        key.append(counts[i][u])  # 错误：整数没有 .append() 方法
```

- **问题**：`key` 被初始化为 `total[i]`（一个数字），你想把后面的计数加进去，但数字不支持 `append`。

- **修正**：你应该把 `key` 初始化为一个**列表**，包含第一个元素，然后再追加。

  ```
  def sort_key(i):
      key = [total[i]]  # 用方括号变成列表
      for u in range(10, 0, -1):
          key.append(counts[i][u])
      return key
  ```

#### 4. 数据结构拼写错误

你代码里一会儿写 `sorce`，一会儿写 `scores`。一定要保持变量名统一，否则会报 `NameError`。

### 自动泊车

#####  1. 核心逻辑错误：`queue.pop()` vs `queue.popleft()`

在 BFS 算法中，我们追求的是“先进先出”（FIFO），所以应该使用 `popleft()`。你使用了 `pop()`，这让你的队列变成了“栈”（后进先出），从而变成了 DFS（深度优先搜索），这无法保证找到的是最短路径。

##### 2. 严重的笔误：`new_x` 与 `new_y`

在 `for` 循环中，你的计算方式有误：

Python

```
new_x, new_y = r + dx, r + dy  # 这里应该是 r + dx, c + dy
```

你把 `c` 写成了 `r`，这导致你的移动逻辑完全乱了。

##### 3. 条件判断错误：`not visited`

你写的是 `if ... and not visited and ...`。 这里的 `visited` 是一个二维列表，它永远为真（True），所以 `not visited` 永远为假（False）。你应该检查的是具体的某个点是否被访问过，即 `not visited[new_x][new_y]`。

##### 4. 循环控制错误

你的 `print(-1)` 写在了 `while` 循环的内部，这意味着只要循环跑了一次没找到目标，程序就会直接打印 `-1` 并结束，而不是等待队列全部遍历完再判断。

### 机器人活动区域

在你的代码这一行： `diff = abs(martix[i][j] - martix[new_x][new_y])`

- **问题所在**：你始终在用**当前邻居** `(new_x, new_y)` 和整个 BFS 的**起点** `(i, j)` 做比较。
- **错误后果**：连通区域的定义通常是“相邻的两个格子高度差不超过 1”。如果起点高度是 5，路径是 `5 -> 6 -> 7`。
  - 按照**正确逻辑**：5 和 6 能连通，6 和 7 也能连通，所以 5, 6, 7 应该在同一个区域。
  - 按照**你的逻辑**：5 和 6 差 1（通过），但 5 和 7 差了 2（失败）。结果 7 就会被孤立，导致面积算小了。

你应该比较的是**当前弹出的节点** `(r, c)` 和它的**邻居** `(new_x, new_y)`。
