# 分批萨

### 环形问题转为线性问题

```python
arr = a + a  # 展开圆形
```

原题描述披萨是**圆形（环形）**的。在环形结构中，第 $N$ 块披萨的旁边紧挨着第 $1$ 块披萨。

- **逻辑难点：** 如果你用数组 `[a1, a2, ..., aN]`，当你吃掉 `a1` 后，剩余的披萨是 `a2` 到 `aN`。如果你想吃 `aN`，它和 `a2` 不在数组的两端，处理起来非常麻烦。
- **展开的意义：** 将数组复制一份接在后面，变成 `[a1, a2, ..., aN, a1, a2, ..., aN]`。
  - 这样，**任何**一段连续的披萨（比如 `aN, a1, a2`）在展开后的数组中都变成了一个**线性的子数组**。
  - 通过 `arr = a + a`，我们将环形问题转化为了线性问题。你只需要在长度为 $2N$ 的数组中，寻找长度正好为 $N$ 的连续区间即可。



#### 疑问

arr = a + a  # 展开圆形我还有一点比较疑惑，就是这个展开就是为了防止最后一个没有“右边”是吧，但是我要是选第一个，我也没有左边呀

---

这是一个非常犀利的观察！你指出的正是“环形数组线性化”中最容易让人困惑的地方：**既然数组变成了线性，那“头”和“尾”的邻接关系该怎么处理？**

其实，`arr = a + a` 这一步，巧妙地通过“冗余”解决了你说的所有边界问题。

假设原本的披萨是 `[A, B, C]`（环状）。 展开后变成：`[A, B, C, A, B, C]`。

- **你的顾虑：** “选第一个（A），左边没有东西。”
- **代码的逻辑：** 在这个展开后的数组中，**每一个位置都可以作为“第一个”被选**。

当你用循环 `for i in range(n)` 枚举起点时：

- 如果你选择从索引 `0` 的 `A` 开始，它对应的“左边”是 `arr[-1]`（即原数组的 `C`），但我们的代码逻辑通过 `dfs(i + 1, i + n - 1, False)` 锁定了**长度为 N 的区间**。
- 这个区间内的 `[i, i + n - 1]` 恰好完整地涵盖了环形披萨的任意一种排列顺序。



### 记忆化搜索模板

> memo 就是记录的这一层dfs的结果，所以memo字典的键就是dfs的参数

```python
# 1. 准备全局或外部的缓存结构
memo = {}

def dfs(state):
    # 2. 终止条件：到达递归边界
    if base_case(state):
        return result
    
    # 3. 记忆化检查：查看是否已经计算过
    if state in memo:
        return memo[state]
    
    # 4. 计算逻辑：进行搜索/状态转移
    res = compute(state) # 可能是多个递归子问题的组合
    
    # 5. 存入缓存：将结果存入字典或数组
    memo[state] = res
    return res
```
