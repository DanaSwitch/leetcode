### 问答

可是在本题中并没有将matrix写入参数呀，而且为什么要设置nonlocal min_dist，而不用设置nonlocal matrix呢，不都是在函数外定义的吗

> 这是一个非常深刻的问题，触及了 Python 作用域（Scope）和变量类型的核心机制。
>
> 你之所以感到困惑，是因为 Python 对**“只读变量”\**和\**“要修改的变量”**的处理方式完全不同。
>
> ------
>
> ### 1. 为什么 `matrix` 不用写进参数，也不用 `nonlocal`？
>
> 在 Python 中，嵌套函数（如 `backtrack`）可以**直接访问**外部函数定义的变量。
>
> - **只读访问（Read-only）：** 如果你只是读取变量的值（例如 `print(matrix)` 或 `val = matrix[i][j]`），Python 会自动从当前作用域向外寻找。既然 `matrix` 在整个递归过程中内容是不变的，我们只需要读取它，所以不需要把它传进参数，也不需要声明 `nonlocal`。
> - **性能考量：** `matrix` 通常很大。如果把它写进参数，虽然 Python 传递的是引用（不会复制整个矩阵），但每一层递归都要在栈帧里多存一个变量名，这在深度递归时会浪费微小的内存和时间。
>
> ------
>
> ### 2. 为什么 `min_dist` 必须要写 `nonlocal`？
>
> 这涉及到一个“赋值即创建”的陷阱：
>
> - **修改冲突：** 在 `backtrack` 内部，我们有一句 `min_dist = min(min_dist, total_dist)`。
> - **Python 的逻辑：** 当 Python 看到你在函数内部给一个变量**赋值**（使用 `=`）时，它默认会认为你想创建一个**新的局部变量**。
> - **报错原因：** 如果不加 `nonlocal`，Python 会在 `backtrack` 内部创建一个同名的局部变量 `min_dist`。但在执行 `min(min_dist, ...)` 时，这个局部的 `min_dist` 还没被赋值，程序就会报错：`UnboundLocalError`（局部变量在赋值前被引用）。
>
> **`nonlocal` 的作用：** 明确告诉 Python：“别在我的函数里创建新变量，去外面找那个已经定义好的 `min_dist`，我要改的就是它！”
>
> ------
>
> ### 3. 为什么 `visited` 数组也不用 `nonlocal`？
>
> 你可能会问：`visited` 也在函数里被修改了（`visited[i] = True`），为什么它不用 `nonlocal`？
>
> - **原地修改（In-place Mutation）：** `visited` 是一个列表（List）。我们没有对 `visited` 变量本身进行重新赋值（没有写 `visited = ...`），我们只是修改了它**内部的值**（`visited[i] = ...`）。
> - **逻辑：** 这种操作不需要 `nonlocal`，因为变量指向的那个“容器”没变，只是容器里的东西变了。
>

> ### 💡 一个终极思考建议
>
> 如果你还是觉得 `nonlocal` 很难记，或者觉得不传 `matrix` 心里不踏实，有一个**“万能作弊法”**：
>
> **把 `min_dist` 存在一个列表里：** `min_dist = [float('inf')]` 然后在函数里写： `min_dist[0] = min(min_dist[0], total_dist)`
>
> 这样 `min_dist` 就变成了“共享容器”，**既不需要 `nonlocal`，也不需要写进参数**，和 `visited` 的逻辑就一模一样了！
