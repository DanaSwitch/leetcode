### 回溯模板

> 找出所有满足条件的解（输出所有可能的组合方案，打印出所有排列）

```python
def backtrack(状态指针, 结果累加器):
    # 【角色 1：终止条件】
    # 思考：进度(累加器)达到预期了吗？或者越界(指针)了吗？
    if 达到目标进度:
        更新全局最优解或保存路径
        return

    # 【角色 2：遍历候选集】
    # 思考：在当前的“状态指针”下，我有多少个合法的选择？
    for 选择 in 当前可用候选集:
        if 满足剪枝条件(如未访问过/数值合规):
            
            # --- 做选择 (更新累加器和状态) ---
            标记已使用 / path.append(选择) / current_sum += cost
            
            # --- 递归 (进入下一层) ---
            # 这里的参数变化体现了“向下探索”
            backtrack(新的指针, 更新后的累加器)
            
            # --- 撤销选择 (回溯的核心) ---
            # 必须要把状态恢复原状，否则会污染同一层的其他分支
            清除标记 / path.pop() / current_sum -= cost
```



### 递归＋记忆化搜索

> 求出问题的最优解（）

```python
# 1. 准备全局或外部的缓存结构
memo = {}

def dfs(state):
    # 2. 终止条件：到达递归边界
    if base_case(state):
        return result
    
    # 3. 记忆化检查：查看是否已经计算过
    if state in memo:
        return memo[state]
    
    # 4. 计算逻辑：进行搜索/状态转移
    res = compute(state) # 可能是多个递归子问题的组合
    
    # 5. 存入缓存：将结果存入字典或数组
    memo[state] = res
    return res
```



### 递归回溯的参数思考

要把递归回溯的参数想清楚，不再“一脸懵逼”，你可以把回溯看作是一场**“有去有回的旅行”**。

你需要带上的行李（参数）永远只有这**四大类**。当你写不出参数时，就按这四个维度挨个问自己：

------

#### 🎒 第一类：地图（不变的全局背景）

**思考：** “我要处理的数据是什么？它在递归过程中会变吗？”

- **参数名：** `data`, `matrix`, `nums`, `target`
- **特点：** 这些参数在递归每一层都**原封不动**地传下去。
- **本题：** 距离矩阵 `matrix`。

#### 📍 第二类：路标（当前的位置/进度）

**思考：** “我现在走到了哪里？我下一步该从哪里开始选，才不会走回头路或重复？”

- **参数名：** `index`, `start`, `curr_node`, `level`
- **特点：** 这是一个**“动词”**，每一层递归都会改变它。
- **技巧：** * 如果是**组合/子集**问题，传 `i + 1`（保证不选重复的）。
  - 如果是**排列/路径**问题，传 `visited` 数组（保证全局唯一）。

#### 🧺 第三类：篮子（目前的战利品/状态）

**思考：** “到这一步为止，我已经选了哪些东西？我已经花了多少成本？”

- **参数名：** `path`, `current_sum`, `current_count`
- **特点：** 记录**“中间结果”**。
- **核心逻辑：** * 如果是**数值**（如 `sum`），递归回来时它会自动变回原来的样子（值传递）。
  - 如果是**列表**（如 `path`），递归回来必须手动 `pop()`（引用传递）。

#### 🏁 第四类：终点线（停止的信号）

**思考：** “什么时候我算‘完成了任务’？或者什么时候‘彻底失败’了？”

- **参数名：** `n`, `k`, `target_sum`
- **特点：** 通常作为 `if` 判断的条件，不一定非要在参数里，有时是全局变量。





### 思考回溯为什么用全局变量

####  深度思考：为什么“共享容器”在回溯中更好？

1. **逻辑清晰**：回溯的本质是“深度优先搜索（DFS）”。它的任务是**遍历**所有可能性。更新一个外部的“全局最优解”更符合直觉。
2. **剪枝方便**：你代码里的 `if cur_sum > min_dist[0]` 只有在 `min_dist[0]` 是全局实时更新的情况下才最有效。
3. **内存友好**：不需要在每一层递归中都维护和传递一个 `res` 变量。





### 例题

> 题目描述
>
> 游乐园有一款互动游戏，游戏开始时会提供n个宝石，每个宝石都一个属性值a1,a2,...an.玩家在游戏前可以挑选x颗宝石，将这些宝石的属性值相乘组成玩家的属性值。游戏玩家需要y点属性值，请帮助游戏玩家计算有多少种计算方式。
>
> 输入描述
>
> 第一行：三个整数n,x,y
>
> - 第一个整数n(0 < n <20)表示宝石总数量。
> - 第二个整数x(0<x <=n)，表示可以选择宝石个数
> - 第三个整数y，表示通过游戏需要的属性值
>
> 第二行：n个整数，a1,a2,...an(-100 < ai < 100)，表示每个宝石的属性值。
>
> 输出描述
>
> 输出一个整数，表示玩家可以通过游戏的挑选方式的数量。
>
> 用例1
>
> 输入
>
> ```none
> 4 2 8
> 2 -3 4 5
> ```
>
> 输出
>
> ```none
> 3
> ```

```python
# 读取输入
n, x, y = map(int, input().split())  # 宝石总数量, 可选宝石数, 属性值
gems = list(map(int, input().split()))

count = 0  # 记录满足条件的方案数

def dfs(start, selected):
    global count
    
    # 已经选够x个了，计算乘积
    if len(selected) == x:
        product = 1  # 初始属性值
        for val in selected:
            product *= val
        if product >= y:
            count += 1
        return
    
    # 2层以上, 从start位置开始往后挑宝石
    for i in range(start, n):
        selected.append(gems[i])   # 选第i个宝石
        dfs(i + 1, selected)       # 继续往后选
        # 能到这一步说明dfs()已经递归返回了, 所有的可能都尝试了一遍
        selected.pop()             # 撤销选择，尝试下一个

dfs(0, [])
print(count)
```
