## 逐步拆解

### ① 枚举一条边，推导另外两个顶点

给定边 P1(x1,y1) → P2(x2,y2)，边向量为 `(dx, dy) = (x2-x1, y2-y1)`。

将该向量旋转 90° 可以得到两种正方形：

**顺时针旋转**（向量变为 `(dy, -dx)` = `(y2-y1, x1-x2)`）：

```
p3 = P1 + 旋转向量 = (x1 + (y2-y1),  y1 + (x1-x2))
                  = (x1 - (y1-y2),  y1 + (x1-x2))   ← 与代码完全对应
p4 = P2 + 旋转向量 = (x2 - (y1-y2),  y2 + (x1-x2))
```

**逆时针旋转**（向量变为 `(-dy, dx)` = `(y1-y2, x2-x1)`）：

```
p5 = (x1 + (y1-y2),  y1 - (x1-x2))   ← 代码中 p5, p6
p6 = (x2 + (y1-y2),  y2 - (x1-x2))
```

### ② 用集合做 O(1) 查找

```python
points = set()   # 存所有点
if p3 in points and p4 in points:   # O(1) 哈希查找
    square_count += 1
```

比枚举 4 点组合再判断快得多。

### ③ 为什么最后 `// 4`？

一个正方形有 4 条边，循环中每条边 `(i, j)` 只被枚举一次（因为 `j > i`），每次枚举恰好在一个方向上命中 → **同一个正方形会被累计 4 次**，所以最终除以 4。
