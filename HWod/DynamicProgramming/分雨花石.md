## 为什么是倒序？

```python
for weight in weights:  # 遍历每个石头
    for j in range(target, weight - 1, -1):  # 倒序
        dp[j] = min(dp[j], dp[j - weight] + 1)  # 拿不拿石头, 拿的话重量减少weight, 石头数量减一 
```



假如说要达到7，有石头5，3，2，1，手动for循环一下

**设定场景：**

- **目标重量 (target)**：7
- **石头重量 (weights)**：[5, 3, 2, 1]
- **初始化**：`dp` 数组长度为 8（索引 0 到 7），初始值 `dp[0] = 0`，其余设为 $\infty$（代表还没凑出来）。

------

### 🟢 第 1 轮：处理石头 5

**遍历范围：从 7 倒序到 5**

1. **j = 7**: `dp[7] = min(inf, dp[7-5] + 1)` $\to$ `dp[2]` 是 $\infty$，保持 `inf`。
2. **j = 6**: `dp[6] = min(inf, dp[6-5] + 1)` $\to$ `dp[1]` 是 $\infty$，保持 `inf`。
3. **j = 5**: `dp[5] = min(inf, dp[5-5] + 1)` $\to$ `dp[0]` 是 0，更新为 **1**。

- **本轮结果**：`[0, inf, inf, inf, inf, 1, inf, inf]`

------

### 🔵 第 2 轮：处理石头 3

**遍历范围：从 7 倒序到 3**

1. **j = 7**: `dp[7] = min(inf, dp[7-3] + 1)` $\to$ `dp[4]` 是 $\infty$，保持 `inf`。
2. **j = 6**: `dp[6] = min(inf, dp[6-3] + 1)` $\to$ `dp[3]` 是 $\infty$，保持 `inf`。
3. **j = 5**: `dp[5] = min(1, dp[5-3] + 1)` $\to$ `dp[2]` 是 $\infty$，保持 **1**。
4. **j = 4**: `dp[4] = min(inf, dp[4-3] + 1)` $\to$ `dp[1]` 是 $\infty$，保持 `inf`。
5. **j = 3**: `dp[3] = min(inf, dp[3-3] + 1)` $\to$ `dp[0]` 是 0，更新为 **1**。

- **本轮结果**：`[0, inf, inf, 1, inf, 1, inf, inf]`

------

### 🟡 第 3 轮：处理石头 2

**遍历范围：从 7 倒序到 2**

1. **j = 7**: `dp[7] = min(inf, dp[7-2] + 1)` $\to$ `dp[5]` 是 1，更新为 **2**（用了 5+2）。
2. **j = 6**: `dp[6] = min(inf, dp[6-2] + 1)` $\to$ `dp[4]` 是 $\infty$，保持 `inf`。
3. **j = 5**: `dp[5] = min(1, dp[5-2] + 1)` $\to$ `dp[3]` 是 1，但 $1 < 1+1$，保持 **1**。
4. **j = 4**: `dp[4] = min(inf, dp[4-2] + 1)` $\to$ `dp[2]` 是 $\infty$，保持 `inf`。
5. **j = 3**: `dp[3] = min(1, dp[3-2] + 1)` $\to$ `dp[1]` 是 $\infty$，保持 **1**。
6. **j = 2**: `dp[2] = min(inf, dp[2-2] + 1)` $\to$ `dp[0]` 是 0，更新为 **1**。

- **本轮结果**：`[0, inf, 1, 1, inf, 1, inf, 2]`

------

### 🔴 第 4 轮：处理石头 1

**遍历范围：从 7 倒序到 1**

1. **j = 7**: `dp[7] = min(2, dp[7-1] + 1)` $\to$ `dp[6]` 是 $\infty$，保持 **2**。
2. **j = 6**: `dp[6] = min(inf, dp[6-1] + 1)` $\to$ `dp[5]` 是 1，更新为 **2**（用了 5+1）。
3. **j = 5**: `dp[5] = min(1, dp[5-1] + 1)` $\to$ `dp[4]` 是 $\infty$，保持 **1**。
4. **j = 4**: `dp[4] = min(inf, dp[4-1] + 1)` $\to$ `dp[3]` 是 1，更新为 **2**（用了 3+1）。
5. **j = 3**: `dp[3] = min(1, dp[3-1] + 1)` $\to$ `dp[2]` 是 1，保持 **1**。
6. **j = 2**: `dp[2] = min(1, dp[2-1] + 1)` $\to$ `dp[1]` 是 $\infty$，保持 **1**。
7. **j = 1**: `dp[1] = min(inf, dp[1-1] + 1)` $\to$ `dp[0]` 是 0，更新为 **1**。

- **最终结果**：`[0, 1, 1, 1, 2, 1, 2, 2]`

------

### ✨ 结论

最终 `dp[7] = 2`。这代表凑成重量 7 最少需要 **2** 块石头（方案是 5+2）。

**你看：** 如果我们在第 4 轮（石头 1）的时候用**正序**：

- 算出 `dp[1]` 变成 1。
- 算出 `dp[2]` 的时候发现 `dp[2-1] + 1 = 2`，而原本 `dp[2]` 是 1，没变。
- 但假如原本没有石头 2，正序就会让 `dp[2]` 变成 2（用了两次石头 1），这就是为什么要**倒序**来保护“旧数据”不被“新数据”污染。
