# 不含101的数

### is_limit的作用

题目：现在给定一个整数区间 [l,r] ，请问这个区间包含了多少个不含 101 的数？

is_limit就是为了防止数超过r



回答你的问题：是不是只有最后一位才受限？

**不是！每一位都可能受限，取决于前面的选择：**

```
n = 1101(二进制)

第0位: is_limit=True  → 只能选 [0, 1]
       ├── 选0: is_limit → False，后续所有位都自由(上界均为1)
       └── 选1: is_limit → True，继续传递
                第1位: is_limit=True  → 只能选 0
                       ├── 选0: is_limit → False，后续自由
                       └── 选1: is_limit → True，继续传递
                                第2位: is_limit=True  → 只能选 0 
                                ← 被限制！
                                       └── 只能选0: is_limit → True
                                                第3位: is_limit=True → 只能选 [0, 1]
```





###  `is_limit and (digit == upper)`

这行代码决定了**下一位是否还受到原始上限 `N` 的约束**。

- **逻辑**：
  - 只有当你填的数 `digit` **等于**当前的上限 `upper`，且上一位已经受限 (`is_limit` 为 `True`) 时，你才需要继续保持受限。
  - 只要你填的数 `digit` 比 `upper` 小，或者之前就已经不受限了，那么你之后就彻底自由了（`is_limit` 变为 `False`）。





### `is_started or (digit == 1)`

这行代码决定了**数字是否已经真正“开始”了**。

- **逻辑**：
  - `is_started` 初始为 `False`。
  - 只要你还没填入过非零数字（即全是前导零），`is_started` 就一直是 `False`。
  - 一旦你在某一位填入了 `1`，那么 `digit == 1` 为真，`is_started` 就会变成 `True`。一旦变成 `True`，它就再也回不去 `False` 了。
