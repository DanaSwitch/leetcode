# 数位DP

## 介绍

数位 DP（Digit Dynamic Programming）是一种专门用来解决**“在一段范围内，统计满足特定条件的数字个数”**的算法技巧。

当题目规模很大（比如 $10^{18}$ 甚至更高）时，直接遍历每一个数会超时，这时数位 DP 的优势就显现出来了。

### 1. 核心思想：按位拆解，逐位决策

数位 DP 的本质是**把一个整数看作一串数字序列**。例如，数 $520$ 可以看作 `[5, 2, 0]`。

我们不再去处理这个数的大小，而是处理“这个数每一位上填什么数字”。通过递归，我们把所有可能的数字组合成一颗“决策树”。

### 2. 为什么需要“动态规划”？

如果不做优化，遍历所有数字的复杂度依然极高。数位 DP 的关键在于**记忆化（Memoization）**。

在搜索过程中，许多子问题的状态是重复的。例如：

- 统计 $1000$ 到 $1999$ 之间满足条件的数。
- 统计 $3000$ 到 $3999$ 之间满足条件的数。

如果这两个区间后三位的限制条件相同，那么它们的结果是可以复用的。我们用 `memo` 字典记录这些状态，下次遇到相同状态时，直接返回答案。

### 3. 常用的“状态”变量

在数位 DP 中，我们通常会传递这几个参数：

- **`index`**：当前正在决定第几位。

- **`is_limit`**：用来防止范围越界的

  > 比如要求在0-520之间选出不含11的数字

  一个布尔值，限制当前位最大能填多少。如果之前填的都是上限（比如 $r=520$，前两位填了 $5, 2$），那么当前位只能填 $0$；如果之前已经比上限小了，那当前位可以填 $0-9$。

  is_limit 表示当前的这一位是否受到限制，is_limit = True表示受到限制，False表示不受限制

- **`is_started`**：处理前导零（比如数字 $5$ 在四位长度下是 $0005$）。

- **`...其他条件`**：根据题目要求设定的状态，比如题目要求不出现 `101`，那我们就需要记录**“前两位是什么”**。



---

is_started 的作用：

**区分“数字没开始”与“数字是 0”**：

- 如果 `is_started` 为 `False`（表示还没开始填有效数字）：当前位填 `0`，意味着**数字还没开始生成**。此时，下一位仍然处于“没开始”的状态。
- 如果 `is_started` 为 `True`（表示已经开始填有效数字）：当前位填 `0`，意味着**这个 `0` 是数字的一部分**（比如 $105$ 中的中间那个 $0$）。



### 4. 解决流程：三步走

1. **转化区间**：将 $[l, r]$ 转化为 `solve(r) - solve(l-1)`。
2. **定义递归**：根据题目条件设计状态（`index`, `last_state`, `is_limit` 等）。
3. **记忆化存储**：利用 `memo` 缓存计算结果。

### 总结

你可以把数位 DP 想象成**“从高位到低位进行的一场深度优先搜索，但通过记忆化手段把所有走过的路都存了下来”**。它将原本需要遍历海量数字的问题，转化为了对数字组成规律的分析，复杂度通常是 $O(\text{位数} \times \text{状态数})$，这对于 $10^9$ 乃至 $10^{18}$ 的输入来说，通常只需几十毫秒即可完成。





## 应用场景

### 通用题目场景

> **“在区间 $[L, R]$ 内，满足特定性质的整数有多少个？”**
>
> （例如：不含数字 4，各位数字之和为偶数，不能出现某种模式等）

### 数位 DP 通用代码模板

```python
def solve(n):
    if n < 0: return 0
    s = str(n)  # 如果是二进制，用 bin(n)[2:]
    memo = {}

    # state: 根据题目需求灵活定义，如 (index, prev_val, is_limit, is_started, ...)
    def dp(index, is_limit, is_started, *args):
        # 1. 递归终止条件：走到底了，返回 1 表示找到一个合法数
        if index == len(s):
            return 1 if is_started else 0 # 根据题目要求调整（是否允许空数）
        
        # 2. 记忆化读取
        state = (index, is_limit, is_started, *args)
        if state in memo:
            return memo[state]
        
        res = 0
        # 3. 确定当前位可以填的数字范围
        upper = int(s[index]) if is_limit else 9 # 如果是二进制则为 1
        
        for digit in range(upper + 1):
            # 4. 判断逻辑：前导零处理和具体限制条件
            new_is_started = is_started or (digit != 0)
            
            # 如果当前位填了数且触犯题目逻辑，跳过
            if not is_valid(digit, *args): 
                continue
            
            # 5. 递归下一位
            new_is_limit = is_limit and (digit == upper)
            res += dp(index + 1, new_is_limit, new_is_started, *next_args)
            
        memo[state] = res
        return res

    return dp(0, True, False, *initial_args)
```

### 关键点总结（填空式学习）

在做任何数位 DP 时，只需思考以下四步：

1. **状态定义 (`state`)**：
   - 除了 `index` 和 `is_limit`，你需要记录什么信息来判断下一步是否合法？
   - *例：本题记录了 `last1` 和 `last2`，因为判断 `101` 需要知道前两位。*
2. **`is_started` 的作用**：
   - 这是为了处理“前导零”。例如在数字 `005` 中，前两个 `0` 不应被视为真正的 `0`，否则会影响对数字性质的判断（比如数字中不能有 `0`）。
3. **`is_limit` 的逻辑**：
   - 这是数位 DP 的灵魂。它限制了当前位能填的最大值。如果不受限，该位可以填 `0-9`（或 `0-1`）；如果受限，只能填 `0` 到 `s[index]`。
4. **递归出口**：
   - 通常返回 `1` 表示找到一个合法数，但在某些特殊题目中，如果不包含前导零，需要额外判断。
