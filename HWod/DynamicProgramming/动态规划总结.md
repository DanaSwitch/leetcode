# 动态规划

## 💡 动态规划通用“三步走”思路

管它题目怎么变，动规的核心就在于**“拆解”**和**“记录”**：

1. **定义状态（找 DP 数组的含义）：** 问自己：`dp[i]` 到底代表什么？是“以 i 结尾的最长长度”？还是“凑够重量 i 需要的最少个数”？
2. **找转移方程（找依赖关系）：** 问自己：我想算出 `dp[i]`，必须先知道哪些前面的值？是 `dp[i-1]`（前一个），还是 `dp[i-weight]`（扣掉当前物品后的状态）？
3. **确定初始值和遍历方向：** 防止数组越界，把最简单的（比如 `dp[0]`）填好；然后根据依赖关系决定是从左往右扫，还是像背包问题那样从右往左扫。

------

## 📋 四大核心模板总结

### 1. 坐标/序列 DP（对应 `shortest_path_between_strings.py`）

**场景：** 字符串对比、矩阵最短路径、编辑距离。

- **核心逻辑：** 建立一个二维表格 `dp[i][j]`，表示 A 串前 i 个和 B 串前 j 个的状态。
- **优化技巧：** 像代码里那样，如果 `dp[i]` 只依赖 `dp[i-1]`，可以用**滚动数组**（`preRow` 和 `curRow`）把空间复杂度从 $O(N^2)$ 降到 $O(N)$。



### 2. 0/1 背包及其变体（对应 `min_stones_dp.py`）

**场景：** 选或不选、恰好凑满、最少个数。

- **模板代码精要：**

  ```python
  for weight in weights:
      for j in range(target, weight - 1, -1): # 必须倒序，防止重复使用同一个物体
          dp[j] = min(dp[j], dp[j - weight] + 1)
  ```

- **关键点：** 目标重量 `target` 通常是总和的一半。

#### 判断是不是01背包

##### 1. 资源总量有限（有一个“大口袋”）

题目中一定会给出一个**总限制**。

- 在 `min_stones_dp.py` 中，这个限制是总重量的一半 `target`。
- 在“小明减肥”题中，这个限制是目标卡路里 $t$。
- **通俗点说：** 你手里有一个容积有限的背包，装满了就再也塞不下了。

##### 2. 物品“不可分割”且“只有一件”（0 或 1）

这是判断“01”背包最关键的点。

- **不可分割：** 你不能只要半个雨花石或者半次运动。要么全拿（1），要么不拿（0）。
- **只有一件：** 每个运动或每块石头在选择列表中只出现一次。
- **对比：** 如果某种物品可以无限次拿取，那是“完全背包”；如果物品可以切开拿，那是“贪心算法”。

##### 3. 每个决定都会消耗资源并产生“贡献”

当你选择一个物品时，会发生两件事：

1. **消耗：** 背包的剩余空间减少了（卡路里增加、重量增加）。
2. **贡献：** 离目标更近了（方案数增加、总价值增加、或凑齐了某个数量）。

##### 🔍 常见的“01 背包”变体套路

在华为 OD 考试中，01 背包通常会变着花样出现：

> **求最大价值**（用 `max`）和**求方案数**（用 `+`）

| **题目问法**                    | **核心转移逻辑**                          | **对应 DP 数组含义**           |
| ------------------------------- | ----------------------------------------- | ------------------------------ |
| **能不能恰好凑到 $T$？**        | `dp[j] = dp[j] or dp[j-cost]` 恰好        | `dp[j]` 是布尔值（True/False） |
| **最少/多需要几个能凑到 $T$？** | `dp[j] = min(dp[j], dp[j-cost] + 1)` 最少 | `dp[j]` 是个数                 |
| **凑到 $T$ 共有多少种方法？**   | `dp[j] = dp[j] + dp[j-cost]`              | `dp[j]` 是方法数               |
| **限定选 $K$ 个凑到 $T$？**     | `dp[i][j] += dp[i-1][j-cost]`             | `dp[i][j]` (增加数量维度)      |

```python
"""
小明有n个可选运动，每个运动有对应卡路里，想选出其中k个运动且卡路里和为t。k，t，n都是给定的。求出可行解数量

输入描述
第一行输入 n t k

第一行输入 每个运动的卡路里 按照空格进行分割
"""
import sys

def solve(n, t, k, sports):
    # dp[i][j]表示用i个运动凑成卡路里j的总方法数
    dp = [[0] * (t+1) for _ in range(k+1)]  # t是列, k是行
    dp[0][0] = 1
    # 遍历每一个运动项目, 每个运动只能用一次
    for sport in sports:
        # 倒序遍历运动个数
        for i in range(k, 0, -1):
            # 倒序遍历卡路里数
            for j in range(t, sport-1, -1):
                # 选不选都加起来
                dp[i][j] += dp[i-1][j-sport]
    print(dp[k][t])


if __name__ == "__main__":
    line = sys.stdin.readline().split()
    n, t, k = map(int, line)
    sports = list(map(int, sys.stdin.readline().strip().split()))
    solve(n, t, k, sports)
```



#### 🚩 警示：什么时候**不是** 01 背包？

如果题目出现以下特征，请立刻回头：

- **物品可以无限用：** 比如换零钱，面值 1 元的硬币有无数个（这是完全背包）。
- **要求相邻不能选：** 比如打家劫舍问题（这是线性 DP）。
- **数据范围极其微小（如 $n < 20$）：** 优先考虑**状态压缩 DP** 或者回溯法，因为它们能处理比背包更复杂（比如有顺序要求）的情况。





### 3. 最长递增子序列 LIS（对应 `book_stacking_lis.py`）

**场景：** 嵌套、堆叠、排队问题。

- **核心逻辑：**  **排序：** 多个维度（长、宽）时，先按一维升序，另一维降序（防止同长时多算）。
  - **二分优化：** 维护一个 `dp` 数组，如果当前数比末尾大就加入，否则用**二分查找**替换第一个不小于它的数。这样复杂度是 $O(N \log N)$。





### 4. 状态压缩 DP（对应 `shortest_delivery_path.py`）

**场景：** 旅行商问题（TSP）、小规模（n < 20）的全排列遍历。

- **核心逻辑：** 用一个整数的**二进制位**（mask）来表示哪些点已经走过。
  - `1 << n`：所有可能的状态组合。
  - `mask | (1 << i)`：标记第 i 个点已访问。
- **口诀：** 看到数据范围里 $n$ 非常小（通常 15 左右），就要往“位运算 + DP”上想。

## 🛠️ 针对你代码的小贴士

- **记忆化递归 (`count_sequences.py`)：** 这是 DP 的另一种写法（自顶向下）。如果你觉得递推很难想，先写递归，然后加个 `memo = {}` 记录算过的值，效果是一样的，而且逻辑更符合直觉。
- **奇偶/状态切换 (`parity_sequence_bit.py`)：** 当题目有“奇偶交替”这种限制时，通常需要定义**多个 DP 数组**（如 `dp_odd`, `dp_even`, `dp_alt`）来分别记录当前结尾数字的不同属性状态。
