`minTotal` 记录的是 **当前正在处理的这个子段（segment）** 内，不是全局变量，在每个字段内是独立的，在**匹配队伍最多的情况下**，所有可能的匹配方案中**最小的相邻队伍差值总和**。



```python
# 计算每段的最小差值和
def getMaxCountMinSum(segment, index, abandon, total, minTotal):
    
    if index >= len(segment):
        if total < minTotal[0]:
            minTotal[0] = total
        return
    # 无论队伍奇偶, 都是两两配对
    getMaxCountMinSum(segment, index + 2, abandon, total + segment[index], minTotal)
    
    # 队伍是奇数个才能执行这一段
    if not abandon and len(segment) % 2 == 0:  # 没有放放弃过且队伍个数是奇数
        getMaxCountMinSum(segment, index + 1, True, total, minTotal) 
```



### minTotal = [float('inf')] 为什么要用列表？

如果你想在函数内部“彻底替换”外部的那个数值，就必须用列表。

### 为什么 `total` 却可以直接写成数？

你看代码里：`getMaxCountMinSum(..., total + segment[index], ...)`。 这里 `total` 确实是数字，为什么不用列表？

- **因为 `total` 不需要“同步”！**
- 每一层递归都有自己的 `total`（代表当前路径的累计和）。
- 我们不希望第一条路径的 `total` 跑到第二条路径去。每一条路径都应该是**独立计算**的。
- 我们只需要在路径结束时，把这个独立的 `total` 和**共享**的 `minTotal[0]` 比一下大小。
