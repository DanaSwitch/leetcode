## 区间判断重叠公式

```python
if max(new_app.start_time, reg.start_time) < min(new_app.end_time, reg.end_time):
```

### 1. 公式的直观含义

假设有两个区间：区间 A $[A_{start}, A_{end})$ 和 区间 B $[B_{start}, B_{end})$。

- `max(A_start, B_start)`：找的是两个起点中**较晚**的那一个（**最晚起点**）。
- `min(A_end, B_end)`：找的是两个终点中**较早**的那一个（**最早终点**）。

**结论：** 如果“最晚起点”比“最早终点”还要**小**，那么这两个区间**一定有交集**。

------

### 2. 为什么能排除“不重叠”的情况？

我们来看看如果不重叠，这个公式会发生什么。

#### 情况一：区间 A 在 区间 B 的左边（完全不碰）

- A: `[09:00, 10:00]`
- B: `[11:00, 12:00]`
- **最晚起点**：`max(09:00, 11:00) = 11:00`
- **最早终点**：`min(10:00, 12:00) = 10:00`
- **判断**：`11:00 < 10:00` ？ **错误（False）！**
- **结果**：判定为**不冲突**。

#### 情况二：区间 A 和 区间 B 刚好首尾相连

- A: `[09:00, 10:00]`
- B: `[10:00, 11:00]`
- **最晚起点**：`max(09:00, 10:00) = 10:00`
- **最早终点**：`min(10:00, 11:00) = 10:00`
- **判断**：`10:00 < 10:00` ？ **错误（False）！**
- **结果**：判定为**不冲突**（符合题目“不包含结束时间点”的规则）。

------

### 3. 为什么这个公式好用？

如果不写这一行，你可能需要写下面这么多代码来覆盖所有重叠情况：

1. A 的起点在 B 内部。
2. A 的终点在 B 内部。
3. A 整个包住了 B。
4. B 整个包住了 A。

而 `max(start) < min(end)` 就像一个**收缩的夹子**：它尝试找出重叠的那一小段。如果夹子的左边（晚起点）跑到了右边（早终点）的右侧，说明夹子夹空了，即没有重叠。



## 自己写错误

### 1. 语法错误（Python 基础）

- **大小写问题**：Python 中的布尔值是 `True` 和 `False`（只有首字母大写），你写成了全大写的 `TRUE` 和 `FALSE`，这会导致程序报错。
- **函数调用缺少括号**：`sys.stdin.readline.split()` 应该是 `sys.stdin.readline().split()`。没有 `()`，你是在引用函数本身而不是在调用它获取字符串。
- **优先级转换**：`priority = part[1]` 得到的还是字符串，在比较大小时（`new_app.priority > reg.priority`）虽然不会报错，但会出现逻辑错误（比如字符串 `"2"` 比 `"10"` 大）。需要转成整数：`int(part[1])`。

### 2. 逻辑错误（核心部分）

- **冲突判定符号写反了**：
  - 你写的：`max(...) > min(...)`
  - 正确的：`max(...) < min(...)`（我们在上一轮讨论过，只有“晚起点”比“早终点”**小**，才有重叠）。
- **`if can_register` 的位置**： 你把这个 `if` 块放在了 `for reg in registered_app` 的**内部**。就像咱们之前讨论的“买地皮”例子，这会导致在还没检查完所有对手时就盲目地注销和添加。它必须移动到 `for reg` 循环的**外面**。
- **查询结果时的变量错误**： `if app.start_time <= query_str.start_time < app.end_time` 这一行里，`query_str` 是个字符串，它没有 `.start_time` 属性。你应该直接用你转换好的整数变量 `query_time`。
