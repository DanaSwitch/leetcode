# FLASH 坏块检测

## 题目解析

这道题本质上是让你模拟一个**“动态连通块增长”**的过程。我们可以把它拆解为“物理场景”和“逻辑规则”来理解。

### 1. 题目到底在干什么？

你可以把 FLASH 介质想象成一张 $m \times n$ 的棋盘。

- **初始状态**：棋盘上所有格子都是干净的（标记为 0）。
- **触发事件**：随着时间推移，有些格子会突然“坏掉”（标记从 0 变为 1）。
- **你的任务**：每当有一个新格子变坏后，你需要立刻数一数，当前棋盘上一共有多少个“坏块”。
- **坏块定义**：如果几个坏掉的格子上下左右连在一起，它们就算作**同一个**坏块（也就是连通分量）。

------

### 2. 输入的含义（结合样例）

输入提供了棋盘大小以及变坏的顺序。

- **第一、二行 ($m=4, n=3$)**：定义了一个 4 行 3 列的矩阵。
- **后续行**：每一行代表一个“时间点”，给出新增异常格子的坐标 $(r, c)$。

**我们看样例的变化过程：**

| **时刻** | **新增位置** | **棋盘变化（1为坏）** | **坏块数量变化**                                         |
| -------- | ------------ | --------------------- | -------------------------------------------------------- |
| **初始** | -            | 全是 0                | 0                                                        |
| **1**    | [1, 1]       | [1, 1] 变 1           | 1 个坏块                                                 |
| **2**    | [2, 2]       | 增加 [2, 2]           | 变成 2 个坏块（它们不相连）                              |
| **3**    | [3, 2]       | 增加 [3, 2]           | 仍是 2 个（因为 [3,2] 和 [2,2] 连上了）                  |
| **4**    | [0, 2]       | 增加 [0, 2]           | 变成 3 个（[0,2] 孤立存在）                              |
| **5**    | [1, 2]       | 增加 [1, 2]           | 变成 1 个（[1,2] 把 [0,2]、[2,2]、[3,2] 全连成一片了！） |

**输出结果**：`[1, 2, 2, 3, 1]`。这正是你每次操作后坏块数量的快照。

------

### 3. 理解难点：为什么是“坏块”？

你需要区分“坏格子”的数量和“坏块”的数量：

- **坏格子**：只是单纯统计矩阵里有多少个 1。
- **坏块**：需要统计有多少个“相互连通的集合”。当一个新变坏的格子填补了两个已有坏块之间的“空隙”时，这两个坏块就会合并成一个，总数反而会减少。

### 总结

这题的核心难点在于：**处理“合并”逻辑**。当新产生的异常点将原本不相连的区域连接起来时，你需要把它们合并统计。

这正是最适合使用**并查集 (Union-Find)** 的场景。如果你打算用其它方法（如 BFS），逻辑上就是：每次操作后，把整个矩阵当作一张图，重新做一次连通性统计，看看能分成多少个孤立的区域。
