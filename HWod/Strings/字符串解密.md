### 对比：字典序 vs ASCII 总和

我们来看两个子串的对决：`"pt"` 和 `"za"`。

| **比较方式**   | **"pt"**                   | **"za"**                  | **胜者**             |
| -------------- | -------------------------- | ------------------------- | -------------------- |
| **ASCII 总和** | $112 + 116 = \mathbf{228}$ | $122 + 97 = \mathbf{219}$ | **`pt`** (总和更大)  |
| **字典序**     | 第一个字母是 `p` (112)     | 第一个字母是 `z` (122)    | **`za`** (`z` > `p`) |



> [-ord(char) for char in x])  为什么这个就可以判断出不同子串的第一个字母位置?



之所以这个列表推导式能判断第一个字母（以及后续字母），是因为 Python 在比较两个列表（List）时，采用的是**“逐位对比”**（Element-wise comparison）的策略。

### Python 列表的比较规则

当你让 Python 比较 `list_A` 和 `list_B` 的大小时，它的内部逻辑如下：

1. **看第一位：** 如果 `list_A[0] < list_B[0]`，则认为 `list_A` 小于 `list_B`。**比赛结束。**
2. **看第二位：** 如果第一位相等，再比较 `list_A[1]` 和 `list_B[1]`。
3. **以此类推：** 只要在某一位分出了胜负，后面的元素统统不再看了。
