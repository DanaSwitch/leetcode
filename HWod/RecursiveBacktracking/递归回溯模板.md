# 模板

## 判断适用条件

### 1. 信号一：关键词是“搜索所有方案”

如果题目要求你找出**所有**满足条件的路径、排列、组合，或者要求你判断“**是否存在**”一种分配方案，这通常就是回溯的信号。

- **题目关键词：** “所有组合”、“所有排列”、“所有子集”、“是否存在一种可能”、“找出所有路径”。
- **本题对应：** “能否把所有得分分成若干组（是否存在这种分组方案）”。

------

### 2. 信号二：决策过程是一棵“多层树”

回溯本质上是在遍历一棵**决策树**。如果你发现解决问题需要一步步做决定，且每一步都有多个选项，那么这就是回溯。

- **第一步：** 第一分钟的分数给谁？（选项：队员A, B, C...）
- **第二步：** 第二分钟的分数给谁？（选项：队员A, B, C...）
- **第 N 步：** ...

如果你能把解题过程画成上面这种不断分叉的树，那它就是回溯。

------

### 3. 信号三：数据范围通常很小

这是最实用的**“考场直觉”**。

回溯算法的复杂度通常是指数级的（例如 $2^n$ 或 $n!$），所以它处理不了海量数据。

- **回溯题的特征：** 数组长度通常在 $20 \sim 50$ 以内（本题 $t \leq 50$）。
- **对比：** 如果数据量是 $10^5$ 级别，那通常得用**贪心**或**动态规划**；如果只有几十个，那大概率就是让你去“搜”（回溯）。

------

### 4. 三大典型题型对照表

为了让你一眼认出它，我帮你整理了回溯的“全家桶”：

| **题型分类**    | **典型题目**                       | **回溯的任务**             |
| --------------- | ---------------------------------- | -------------------------- |
| **组合/排列类** | 全排列、电话号码的字母组合         | 挨个试每个字母或数字的组合 |
| **路径搜索类**  | 迷宫寻路、单词搜索、N皇后          | 试着走每条路，撞墙了就回头 |
| **资源分配类**  | **本题（得分分配）**、火柴拼正方形 | 试着把资源放进不同的桶里   |



## 1. 核心模板

```python
def dfs(状态) {
    if (终止条件) { 收集结果; return; }

    for (每一种可能的选择) {   // ← 横向，枚举同层分支
        做选择;
        dfs(下一层状态);       // ← 纵向，深入下一层
        撤销选择;
    }
}
```



```python
def backtrack(state, options):
    # 1. 终点判断：如果满足停止条件
    if is_solution(state):
        return True # 或者把结果存入全局变量
    
    # 2. 遍历所有选项：尝试每一种可能性
    for option in options:
        # 3. 剪枝：如果当前选项明显不合适，跳过
        if not is_valid(option, state):
            continue
        
        # 4. 做出选择：把选项加入当前状态
        make_choice(option, state)
        
        # 5. 继续探索：进入下一层递归（向后走一步）
        if backtrack(next_state, options):
            return True # 只要下层通了，我这就通了
        
        # 6. 撤销选择：回溯（重要！退回到之前的状态）
        undo_choice(option, state)
    
    # 7. 彻底失败：当前层所有选项都试过了，无解
    return False
```

## 2.三个核心部分的含义

#### 第一步：终止条件 (Base Case)

就像迷宫的出口。如果没有这一行，递归会无限循环下去导致内存溢出。通常是判断“所有数字都分完了”或者“格子都填满了”。

#### 第二步：做出选择 (Make Choice)

这是你决定走哪条路的地方。在你的题目里，就是`buckets[i] += scores[idx]`，即“把这分钟的分数给这个队员”。

#### 第三步：撤销选择 (Undo Choice / Backtracking)

**这是回溯的灵魂。** 它的作用是**清理现场**。

因为你在递归时修改了公共的状态（比如队员的得分列表），如果你不把它减回去，等你想尝试“把分数给下一个队员”时，之前的得分还留在里面，逻辑就乱了。

## 3. 如何把模板应用到本题？

我们可以把你的代码和模板一一对应：

| **模板动作** | **本题对应代码**                        | **解释**                             |
| ------------ | --------------------------------------- | ------------------------------------ |
| **终止条件** | `if idx == len(scores): return True`    | 所有分钟都分完了，成功！             |
| **遍历选项** | `for i in range(k):`                    | 尝试分给 $k$ 个队员中的每一个。      |
| **剪枝**     | `if buckets[i] + scores[idx] > target:` | 如果塞进这个队员会超分，跳过。       |
| **做出选择** | `buckets[i] += scores[idx]`             | 假定分给第 $i$ 个队员。              |
| **继续探索** | `if backtrack(idx + 1): return True`    | 问后面的人：剩下的能搞定吗？         |
| **撤销选择** | `buckets[i] -= scores[idx]`             | 后面搞不定？把分拿回来，试下一个人。 |
