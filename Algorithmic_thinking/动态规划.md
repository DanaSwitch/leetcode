# 动态规划

简单来说，**动态规划（Dynamic Programming，简称 DP）既是一种算法思想，也有一套相对固定的解题建模套路。**

如果用一句话概括 DP 的核心，那就是：**“拆解问题，记住过去。”** 它通过把原问题分解为相对简单的子问题，并保存子问题的结果，从而避免了重复计算。

------

### 1. 动态规划的“思想”核心

很多初学者容易把 DP 和递归混淆。其实 DP 的精髓在于以下三个概念：

1. **最优子结构**：大问题的最优解包含子问题的最优解。
2. **重叠子问题**：在计算过程中，同一个子问题会被反复计算（DP 就是为了消灭这种重复）。
3. **状态转移**：寻找“现在”和“过去”之间的逻辑桥梁。

------

### 2. 动态规划的“解题模板”（五部曲）

虽然没有万能的代码模板，但有一个放之四海而皆准的**思考框架**：

1. **确定 dp 数组及下标的含义**：`dp[i]` 代表什么？（这是最关键的一步）
2. **确定递推公式**：`dp[i]` 怎么由之前的 `dp[j]` 推导出来？（又称状态转移方程）
3. **dp 数组如何初始化**：起点在哪里？
4. **确定遍历顺序**：是从前向后，还是从后向前？
5. **举例推导 dp 数组**：手动模拟一遍，看看对不对。

------

### 3. 以经典问题为例：爬楼梯

**题目描述**：假设你正在爬楼梯。需要 $n$ 阶你才能到达楼顶。每次你可以爬 $1$ 或 $2$ 个台阶。你有多少种不同的方法可以爬到楼顶呢？

#### Python3 实现

动态规划通常有两种写法：**自底向上（迭代）** 和 **自顶向下（递归+备忘录）**。

#### 方法 A：自底向上（标准 DP 表法）

这是最常用的“模板”方式。

Python

```
def climbStairs(n: int) -> int:
    if n <= 2:
        return n
    
    # 1. 确定 dp 数组含义：dp[i] 表示爬到第 i 阶的方法数
    dp = [0] * (n + 1)
    
    # 2. 初始化
    dp[1] = 1
    dp[2] = 2
    
    # 3. 确定遍历顺序并推导
    # 递推公式：dp[i] = dp[i-1] + dp[i-2]
    # 解释：到达第 i 阶只有两种可能：从 i-1 迈一步，或者从 i-2 迈两步
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
        
    return dp[n]
```

#### 方法 B：状态压缩（优化空间）

你会发现计算 `dp[i]` 只依赖前两个数，所以不需要整个数组。

Python

```
def climbStairs(n: int) -> int:
    if n <= 2: return n
    
    a, b = 1, 2
    for i in range(3, n + 1):
        # 滚动更新
        a, b = b, a + b
    return b
```

------

### 4. 什么时候该用 DP？

如果你发现一个问题满足以下特征，就可以考虑 DP：

- **求最值**（最大利润、最短路径、最少硬币数）。
- **求可行性**（能否凑成总和、能否跳到终点）。
- **求方案总数**。

**它的对手：** * 如果问题没有重叠子问题（每个分支都是全新的），那叫**分治**（如归并排序）。

- 如果每一步只选当下看起来最好的，不回头看，那叫**贪心**。

------

### 总结

动态规划不是靠“背”代码练出来的，而是靠**“定义状态”**练出来的。当你能准确写出 `dp[i]` 代表什么，并能像写数学归纳法一样写出递推公式时，你就掌握了它。

# 背包问题

“背包问题”（Knapsack Problem）是动态规划领域里最经典、最能体现“状态转移”精髓的模型。

简单来说，它的核心矛盾是：**资源有限（空间），而欲望无限（价值），你必须在有限的负重下，选择收益最高的组合。**

------

### 1. 0-1 背包问题：最基础的型

**场景描述**：

你有一个容量为 $W$ 的背包，面前有 $n$ 件物品。每件物品 $i$ 都有自己的重量 $weight[i]$ 和价值 $value[i]$。

**规则**：每件物品**只有一件**，你只有两个选择：**选（1）** 或者 **不选（0）**。

------

### 2. 动规五部曲拆解

我们以这个例子为例：背包容量 $4kg$，物品如下：

| 物品 | 重量 | 价值 |

| :--- | :--- | :--- |

| 物品 0 | 1kg | 15元 |

| 物品 1 | 3kg | 20元 |

| 物品 2 | 4kg | 30元 |

#### 第一步：确定 dp 数组含义

我们需要一个二维数组 `dp[i][j]`。

- **$i$**：表示考虑到第 $i$ 个物品。
- **$j$**：表示当前背包剩余的容量。
- **`dp[i][j]`**：表示在容量为 $j$ 时，从前 $i$ 个物品中能选出的**最大总价值**。

#### 第二步：确定递推公式

对于物品 $i$，你有两个选择：

1. **不放物品 $i$**：那么最大价值就等于只考虑前 $i-1$ 个物品时的价值。
   - `dp[i][j] = dp[i-1][j]`
2. **放物品 $i$**：前提是背包得装得下（$j \ge weight[i]$）。如果你放了它，那么价值就是：物品 $i$ 的价值 + 扣除掉 $weight[i]$ 后剩余空间能装的最大价值。
   - `dp[i][j] = value[i] + dp[i-1][j - weight[i]]`

**最终公式：** `dp[i][j] = max(dp[i-1][j], value[i] + dp[i-1][j - weight[i]])`

#### 第三步：初始化

- 如果背包容量 $j=0$，价值全是 $0$。
- 如果只有物品 0，只要 $j \ge weight[0]$，价值就是 $value[0]$。

------

### 3. Python3 代码实现（二维数组版）

```python
def knapsack_01(weights, values, capacity):
    n = len(weights)
    # 初始化 dp 数组 (n 行 capacity+1 列)
    dp = [[0] * (capacity + 1) for _ in range(n)]

    # 初始化第一行 (只放物品 0)
    for j in range(weights[0], capacity + 1):
        dp[0][j] = values[0]

    # 开始遍历
    for i in range(1, n):          # 遍历物品
        for j in range(capacity + 1): # 遍历背包容量
            if j < weights[i]:
                # 容量不够，只能不放物品 i
                dp[i][j] = dp[i-1][j]
            else:
                # 比较：不放 i vs 放 i
                dp[i][j] = max(dp[i-1][j], dp[i-1][j - weights[i]] + values[i])
    
    return dp[n-1][capacity]

# 测试数据
weights = [1, 3, 4]
values = [15, 20, 30]
capacity = 4
print(f"最大价值为: {knapsack_01(weights, values, capacity)}") # 输出 35 (选物品0和1)
```

------

### 4. 背包问题的“变种”家族

0-1 背包只是开端，实际刷题中你会遇到它的各种变体：

1. **完全背包**：每件物品有**无数件**。
   - *区别*：遍历顺序变一下即可。
2. **多重背包**：每件物品有**有限件**（比如物品 A 有 3 件）。
3. **分组背包**：物品被分成几组，每组里**只能选一个**。

------

### 💡 为什么说它是“动态规划”？

如果不查表（DP），我们要解决这个问题可能需要暴力搜索所有组合（$2^n$ 复杂度）。

但有了 `dp` 数组，我们其实是在**填表**。每填一个空格，我们都在利用之前已经算好的最优解。



要把二维数组优化成一维数组，我们需要一点“空间想象力”。

在二维 DP 中，计算 `dp[i][j]` 只需要用到 **上一行** 的数据：`dp[i-1][j]`（正上方）和 `dp[i-1][j - weight[i]]`（左上方）。既然只看上一行，我们其实可以只用一个一维数组 `dp[j]`，不断地用“新值”覆盖“旧值”。

### 1. 核心变化：倒序遍历

这是最关键的一点：**在 0-1 背包的一维优化中，内层循环（背包容量）必须从大到小遍历。**

- **为什么要倒序？**

  如果我们从小到大遍历，计算后面的 `dp[j]` 时，前面的 `dp[j - weight[i]]` 可能已经被当前物品 $i$ 更新过了（变成了“这一行”的新值）。这相当于你把同一个物品放进去好几次，变成了“完全背包”。

  **倒序遍历**保证了我们在计算 `dp[j]` 时，用到的 `dp[j - weight[i]]` 还是“上一行”留下来的旧值。

------

### 2. Python3 代码实现（一维滚动数组）

这种写法在算法面试中更常见，因为它空间复杂度从 $O(N \times W)$ 降到了 $O(W)$。

```python
def knapsack_01_optimized(weights, values, capacity):
    # 1. 初始化一维 dp 数组，dp[j] 表示容量为 j 时的最大价值
    dp = [0] * (capacity + 1)
    
    # 2. 遍历物品
    for i in range(len(weights)):
        # 3. 倒序遍历背包容量！
        # 只要容量 j >= 当前物品重量，就尝试更新
        for j in range(capacity, weights[i] - 1, -1):
            # 递推公式：dp[j] 是原本的 dp[j] (不放) 
            # 和 dp[j - weights[i]] + values[i] (放) 之间的最大值
            dp[j] = max(dp[j], dp[j - weights[i]] + values[i])
            
    return dp[capacity]

# 测试数据
weights = [1, 3, 4]
values = [15, 20, 30]
capacity = 4
print(f"优化后的最大价值为: {knapsack_01_optimized(weights, values, capacity)}")
```

------

### 3. 对比总结

| **维度**    | **空间复杂度**  | **遍历顺序**     | **核心逻辑**                                    |
| ----------- | --------------- | ---------------- | ----------------------------------------------- |
| **二维 DP** | $O(N \times W)$ | 正序/倒序均可    | 明确记录“前 $i$ 个物品”的状态，好理解但费内存。 |
| **一维 DP** | $O(W)$          | **容量必须倒序** | 极大地节省空间，是 0-1 背包的进阶标准写法。     |

------

### 4. 动态规划的“玄学”调试法

如果你在写 DP 时发现结果不对，通常是以下三个原因：

1. **初始化错了**：比如求最小值却初始化为 $0$（应该初始化为无穷大）。
2. **遍历顺序反了**：比如 0-1 背包写成了正序。
3. **状态定义不准**：`dp[j]` 的含义在脑子里模糊了。

**这就是动态规划的魅力：代码可能只有十几行，但每一行都凝聚了深厚的逻辑推导。**
